# GozonApplication
Приложение маркетплейса с реализованной фронт-энд частью.

## Запуск и подключение
Для запуска используйте команду
```shell
docker compose up -d --build
```
Я искренне надеюсь, что в этот раз миграции подключились корректно и не нужно запускать их вручную перед compose, 
но если все-таки я ошибся и надо, то вот команды
```shell
dotnet ef migrations add Initial --project src/OrdersService/OrdersService.Infrastructure --startup-project src/OrdersService/OrdersService.Host
```

```shell
dotnet ef migrations add Initial --project src/PaymentsService/PaymentsService.Infrastructure --startup-project src/PaymentsService/PaymentsService.Host
```

Чтобы просмотреть сваггер и UI используйте следующие порты
```shell
http://localhost:6001 - Order Service
http://localhost:6002 - Payment Service
http://localhost:9000 - API Gateway
http://localhost:3000 - Frontend Application
```

## Реализация
Приложение я реализовал в виде двух отдельных микросервисов — заказов и платежей, каждый со своей базой данных и логикой. 
Для реализации платежей я выбрал асинхронное взаимодействие через Kafka. 
Сервис заказов отправляет сообщения о создании заказа в очередь, сервис платежей их обрабатывает и возвращает результат обратно. 
Снаружи всё это скрыто за API Gateway.

## Гейтвей
Я пытался разобраться с NGINX и YARP чтобы сделать простой гейтвей, но там были вечные проблемы (если бы сразу делал вручную сдал бы еще вчера), поэтому сделал гейтвей полностью вручную на основе микросервисов и со слоистой архитектурой.

## Фронтэнд
Реализован на Vite с применением фреймворка Vue.js на языке программирования TypeScript. Получение API запросов вынесено в отдельные файлы. Реализованы в UI все функции из основной части приложения. Есть переключение темы, но отображение стабильнее работает на темной теме, советую пользоваться ей.
